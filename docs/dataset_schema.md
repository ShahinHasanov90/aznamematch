# Dataset schema (v1.0)

Generated by `aznamematch generate`. Source models: `generate/surface.py` (surface forms),
`generate/pairs.py` (labeled pairs). Provenance rule: `docs/rules/provenance-metadata.md`.

## Files

| Path | Committed? | Contents |
|------|-----------|----------|
| `data/full/surfaces.parquet` | no (regenerable) | every surface form |
| `data/full/pairs.parquet` | no (regenerable) | the full labeled-pair set |
| `data/sample/identities_sample.csv` | yes | first N canonical identities |
| `data/sample/surfaces_sample.csv` | yes | first N surface forms |
| `data/sample/pairs_sample.csv` | yes | first N labeled pairs (shuffled mix) |
| `data/adversarial/homoglyph_v1/homoglyphs.parquet` | yes | frozen homoglyph attack set |
| `data/adversarial/homoglyph_v1/homoglyphs_sample.csv` | yes | sample of the above |

The full parquet is git-ignored because it is byte-reproducible from the seed in
`configs/generation.yaml`; the committed CSV samples make the repo inspectable with zero
setup, and the adversarial set is committed because it is frozen and versioned.

## Identity (`generate/canonical.py::Identity`)

`canonical_id` (E######), `entity_type` (person|org), `name_origin_group`, `canonical`
(AZ-Latin full form), `gender` (m|f|""), `given`, `patronymic`, `family`, `family_root`,
`patronymic_father`, `canonical_style`, `org_tokens`, `dob` (yyyy-mm-dd|""), `id_number`,
`family_variants` (`text|style|transform;...`), `russified_patronymic`.

## Surface form (`generate/surface.py::SurfaceForm`)

| Field | Values / meaning |
|-------|------------------|
| `canonical_id` | ground-truth identity id |
| `entity_type` | `person` \| `org` |
| `surface` | the rendered string |
| `script` | `latn-az` \| `cyrl` \| `latn-translit` |
| `translit_standard` | `ICAO` \| `GOST` \| `ISO9` \| `BGN_PCGN` \| `ALA_LC` \| `ad_hoc` \| `none` |
| `corruption_types` | `;`-joined subset of Phase-3 types (empty = clean) |
| `suffix_transform` | `none` \| `national` \| `patronymic` \| `assimilation` |
| `name_origin_group` | fairness slice (e.g. `russified`, `national_zade`, `organization`) |
| `given`, `patronymic`, `family`, `org_tokens` | structured AZ-Latin components |
| `is_homoglyph`, `homoglyph_codepoints` | always `False`/empty in the core set |

## Labeled pair (`generate/pairs.py::Pair`)

| Field | Values / meaning |
|-------|------------------|
| `pair_id` | `P######` (pos) / `N######` (easy neg) / `H..#####` (hard neg) |
| `id1`, `id2` | the two members' identity ids |
| `surface1`, `surface2` | the two surface strings |
| `label` | `1` (same entity) \| `0` (different) |
| `script_pair` | `AZ-RU` \| `AZ-EN` \| `RU-EN` \| `same` (AZ=`latn-az`, RU=`cyrl`, EN=`latn-translit`) |
| `hardness` | `easy` \| `hard` |
| `hard_negative_type` | `none` \| `token_swap` \| `generation_collision` \| `surname_collision` \| `one_edit` |
| `scripts`, `standards`, `corruption_types`, `suffix_transforms`, `name_origin_groups` | `;`-joined union of both members' provenance |
| `is_homoglyph` | `False` in the core set |

### Pair composition

Counts and ratios come from `pairs.*` in the config: `positives_per_identity`,
`neg_per_pos`, `hard_negative_fraction`, `hard_negative_mix`, and `script_pair_targets`.

Positives **and** easy negatives are **stratified** across script-pair cells to
`script_pair_targets` (default AZ-RU 0.30 / RU-EN 0.30 / AZ-EN 0.22 / same 0.18), and the
Cyrillic surface pool is enlarged via `noise.cyrillic_extra_corrupted` so the AZ-RU cell is
large, diverse, and label-balanced. **These cell proportions are engineered for evaluability,
NOT estimates of real-world script-pair frequency** — see docs/methodology.md "Engineered
cross-script distribution". Hard negatives are *synthesized* from the component banks (not
drawn from the 500 canonical identities), are AZ-Latin (`same` script cell), and carry
synthetic ids; slice them by `hard_negative_type`.

## Freeze

The core pair set (`data/full/pairs.parquet` + committed samples) is **v1.0**. The
adversarial directory (`data/adversarial/`) is versioned separately so obfuscation sets can
grow without disturbing v1.0.
